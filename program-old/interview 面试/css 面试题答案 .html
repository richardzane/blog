    css 盒子模型

    RichardZane

    # css 盒子模型

    • css 盒子模型包括 w3c 标准盒子模型和 ie 盒子模型。
    • w3c 标准盒子模型的宽高仅仅包括 content 内容，并不包括 padding 内边距和 border 边框；而 ie 盒子模型的宽高包括了 content 内容、padding 内边距和 border 边框。
    • 可以通过设置 box-sizing 的值为 border-box，将 w3c 标准盒子模型转化为 ie 盒子模型；box-sizing 的值默认是 content-box。

    # css height 和 line-height 的区别

    • 如果只给元素设置了 line-height，那么元素的高度等于行高*文字的行数。
    • 如果只给元素设置了 height，那么元素的高度是固定不变的。

    # css 选择器

    • id 选择器
    • 类选择器、属性选择器、伪类选择器
    • 标签选择器、伪元素选择器
    • 后代选择器、子代选择器、相邻选择器、通配符选择器

    # css 可以继承的属性

    • font-size 文字大小；
    • color 文字颜色；
    • line-height 行高；
    • visibility 可见性；
    • cursor 鼠标形状；

    # css 样式权重（优先级）

    • !important 的权重是 ∞；
    • 行内样式的权重是 1000；
    • id 选择器的权重是 0100；
    • 类选择器、属性选择器、伪类选择器的权重是 0010；
    • 标签选择器、伪元素选择器的权重是 0001；
    • 后代选择器、子代选择器、相邻选择器、通配符选择器的权重是 0000；

    # css 画三角形

    比如画一个向下的三角形，首先把元素的宽和高都设置为0，然后设置 border，比如给一条 8px 的实线的边框，最后设置 border-color 边框颜色，除了给上边框设置具体的颜色之外，其他的边框颜色都设置为 transparent 透明。

    # css 实现不给宽高的元素水平垂直居中的方法

    我经常使用的方法主要有两种：一种是使用 flex 布局，设置父级元素的 display 的值为 flex，设置 justify-content 的值为 center，设置 align-items 的值为 center；另一种是使用 position 定位，子绝父相，设置父级元素的 position 的值为 relative，设置子元素的 position 的值为 absolute，设置 top 的值为 50%，设置 left 的值为 50%，设置 transform: translate 的值为（-50%,-50%）。

    # css display 的值和作用

    display 的值比较多。一个是 none，可以用来隐藏元素；一个 block，可以用来显示元素，或者将元素转换成块级元素；一个 inline-block，可以用来将元素转换成行内块元素；一个是 inline，可以用来将元素转换成行内元素；还有 flex，可以用来弹性布局。

    # css BFC

    bfc 的全称是 block formatting context，就是块级格式化上下文。我把它理解为页面上的一个独立的隔离的容器，在这个容器里面的子元素不管怎么布局，都不会影响到容器外面的其他元素。

    我了解到的触发 bfc 的方法包括：让 float 的值不是 none；让 overflow 的值不是 visible；让 display 的值是 inline-block 或者 table-cell；让 position 的值是 fixed 或者 absolute。

    # css 解决高度塌陷问题（清除浮动方式）

    • 一种是在浮动的子元素的最后，添加一个额外的元素，设置 clear 的值为 both；
    • 一种是设置父级元素的 overflow 的值为 hidden，触发 bfc；
    • 一种是使用 ::before 单伪元素；
    • 还有一种是结合使用 ::before 和 ::after 双伪元素。
    .clearfix::before {
	content: "";
	display: block;
	height: 0;
	clear: both;
	visibility: hidden;
}

    .clearfix::before, .clearfix::after {
	content: "";
	display: table;
}
.clearfix::after {
		clear: both;
}

    # css 解决外边距合并问题

    • 父子元素的外边距合并，给父级元素设置 padding-top 。
    • 相邻元素的外边距合并，给其中一个元素设置 margin。

    # css font-size 字体大小使用奇数还是偶数

    一、偶数。浏览器渲染偶数大小字体的文字，会在网页中表现的更加对称和丰满，奇数显得略微单薄。比如渲染“中”字。

    二、另外，移动端设计稿一般是 750px，但是调试通常以 375px 为标准，偶数方便做除法计算。

    # position 的值以及作用

    positon 的值包括：

    • static：静态定位，默认定位，依附文档流。
    • fixed：固定定位，相对于浏览器窗口的定位，脱离文档流。
    • relative：相对定位，相对于自身的定位，依附文档流。如果设置了四个值，则只有 top 和 left 生效。
    • absolute：决定定位，相对于父级元素的定位，脱离文档流。如果设置了四个值，则都生效，元素展开。

    # 实现双飞翼布局

    一、概念：双飞翼布局就是左右宽度固定，中间宽度动态变化。

    二、实现：让中间、左边、右边依次向左浮动，中间宽度 100%，左边宽度 200px，向左移动 -100%，右边宽度 200px，向左移动 200px：

    <div class="demo">
			<div class="center">
				center
			</div>
			<div class="left"></div>
			<div class="right"></div>
		</div>

    * {
				margin: 0;
				padding: 0;
			}

			.demo {
				width: 100%;
				height: 500px;
			}

			.demo>div {
				float: left;
				height: 100%;
			}

			.center {
				box-sizing: border-box;
				width: 100%;
				padding: 0 200px;
				background-color: red;
			}

			.left {
				width: 200px;
				margin-left: -100%;
				background-color: green;

			}

			.right {
				width: 200px;
				margin-left: -200px;
				background-color: blue;
			}

    # reset.css 和 normalize.css 文件的作用

    一、reset.css 文件用来重置元素的样式。

    二、normalize.css 文件用来增强元素在不同浏览器上渲染的一致性，同时它也重置了部分元素的样式。

    # css 精灵图

    一、使用：

    结合设置 background-image 和 background-position，注意这里的位移是负数。

    二、优缺点：

    • 优点：节约网络资源。把许多小图片集中设计成一张大图片，浏览器只需加载一次大图片。
    • 缺点：改动比较麻烦。把许多小图片集中在一起，如果要调整某个的位置，就会影响其他的位置，只能重新制作大图片。

    # css display:none; 和 visibility:hidden; 的区别

    一、它们都可以用来隐藏元素。

    二、display:none; 隐藏的元素在文档中是不占据位置的，visibility:hidden；隐藏的元素在文档中是占据位置的。

    三、display:none；隐藏元素会造成回流和重绘，visibility:hidden; 隐藏元素只会造成重绘。

    # opacity 和 rgba

    • opacity 和 rgba 都可以用来设置元素的透明度。

    • opacity 的取值范围是 0-1；rgba 中的 r 表示 red 红色，g 表示 green 绿色，b 表示 black 黑色，它们是用来设置颜色的，取值范围是 0-255（或者百分数），rgba 中的 a 是用来设置透明度的，取值范围是 0-1。

    • 子元素会继承父元素设置的 opacity，但是不会继承父元素设置的 rgba。

    # 回流（重排）和重绘

    一、参考文章：

    • https://blog.csdn.net/a419419/article/details/90402561
    • https://segmentfault.com/a/1190000017329980

    二、大致理解的浏览器渲染过程如下：

    • 渲染引擎通过 html 解析器解析 html，生成 dom tree（文档对象模型树）
    • 渲染引擎通过 css 解析器解析 css，生成 cssom tree（css 对象模型树）
    • 将 dom tree 和 cssom tree 结合起来，生成 render tree（渲染树）
    • 浏览器根据渲染树进行回流，得到节点的几何信息
    • 浏览器根据渲染树和几何信息进行重绘，得到节点的像素信息。

    三、元素大小的改变、位置的改变会导致回流重绘，元素文字颜色、背景颜色的改变只会导致重绘。

    四、回流一定引起重绘，重绘不一定引起回流。
